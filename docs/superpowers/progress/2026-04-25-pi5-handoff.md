## 2 · How Pi differs from iMac, code-wise

The repo is shared. Behavior diverges via:

### A. `PI_MODE` env gate

`bird_pipeline_v3.py` line 193: `PI_MODE = os.environ.get("PI_MODE", "0") == "1"`

When `PI_MODE=1`:
- Imports `pipeline.model_registry.build_default_registry` + `pipeline.pi_classifier.PiClassifier` instead of `pipeline.classifier.SmartClassifier`
- Calls `build_default_registry(..., exclude_hailo=True)` — pipeline-side registry excludes Hailo classifier candidates because they'd conflict with the Hailo YOLO detector for the single Hailo-8L vdevice slot
- Imports `pipeline.hailo_detector.HailoDetector` instead of `pipeline.detector.BirdDetector`
- Loads YOLOv8s from `/usr/share/hailo-models/yolov8s_h8l.hef` (override via `PI_YOLO_HEF` env)

When `PI_MODE` unset (iMac):
- Uses `SmartClassifier` (yard + AIY via Coral) per-camera decision tree
- Uses `BirdDetector` (YOLOv8n via ONNX+CoreML)

### B. Pi-only modules

| File | Purpose |
|---|---|
| `pipeline/hailo_detector.py` | YOLOv8 on Hailo via HailoRT Python API (VDevice + InferVStreams + per-class NMS list parser). 25ms / 39 FPS @ 1280×720 |
| `pipeline/hailo_classifier.py` | Generic Hailo classifier wrapper (used by Lab upload-test) |
| `pipeline/model_registry.py` | CandidateModel dataclass + ModelRegistry. `build_default_registry(exclude_hailo: bool)`. 5 candidates: aiy_onnx, resnet50_hailo, yolov8s_hailo, yolov6n_hailo, flagship_pending. Honors `PI_CLASSIFIER` env at startup |
| `pipeline/pi_classifier.py` | PiClassifier wrapping ModelRegistry with SmartClassifier-compatible interface |
| `dashboard/pi_dash.html` | Pi-native dashboard (4 themes, Model Lab, Recent Classifications strip, no live-feed panel) |

### C. PI_MODE-gated branches in shared files

`dashboard/api.py`:
- `_pi_active_classifier()` → reads `PI_CLASSIFIER` from `~/.bird-observatory-env`
- `_pi_update_env_classifier(name)` → writes env file + `systemctl --user restart bird-pipeline`. Real model swap via service restart (not in-process hot-swap)
- `/api/models/list` returns `current = _pi_active_classifier()` on Pi (env-derived) vs `_lab_registry.current_name` on iMac
- `/api/models/switch` Pi path goes through `_pi_update_env_classifier`; iMac path is in-process registry switch
- `/` route serves `pi_dash.html` if PI_MODE=1, else `index.html`
- Two registries on dashboard:
  - `_get_model_registry()` (lab — includes hailo for upload-test)
  - `_get_pipeline_view_registry()` (excludes hailo on Pi — used by /api/models/list and /switch validation)

`pipeline/hires_ring.py`:
- Identical code on both
- Activated on Pi (`PIPELINE_HIRES_RING=authoritative` in env file)
- Inactive on iMac (env unset → `hires_ring=None`)

## 6 · Dangers / things that have bitten us

1. **Hailo VDevice exhaustion (`HAILO_OUT_OF_PHYSICAL_DEVICES(74)`).** Hailo-8L has ONE vdevice slot. Two `VDevice()` instances in one Python process = error. Mitigation in current code: `build_default_registry(exclude_hailo=True)` on the pipeline side. Multi-model proper fix (Path 1) is the playbook's solution.

2. **Coral USB on Pi.** There IS no Coral on Pi. Don't import `pycoral` or `yard_classifier`. The lazy-load in `pipeline/classifier.py` (line 52) handles this; never reverse it.

3. **Hardcoded paths.** Most have been fixed (`shutil.which("ffmpeg")`, `Path.home()`), but if you copy any iMac code over, double-check for `/Users/vives/...` or `/usr/local/bin/...` references.

4. **Pi's classifications.db is ITS OWN DB.** Don't merge with iMac's. Pi has been classifying ~since 2026-04-24 morning, has ~500+ rows, 0 reviews so far. iMac has 155K rows, 1.8K reviews.

5. **HLS recorder restart hazard.** If you `kill -9` a hailo-using process mid-inference, the Hailo PCIe driver holds the device "busy" for a few seconds. Same applies to ffmpeg restarts. Use `systemctl --user restart` (graceful) not `pkill -9`.

6. **The bird-pipeline service uses `Restart=always` with `RestartSec=10`.** If it crashes 3× in quick succession, systemd may rate-limit. Check with `systemctl --user status bird-pipeline -n 50`.

7. **rsync from iMac stomps Pi-local edits.** Always edit in `/Users/vives/bird-classifier/` on iMac, then rsync to Pi. Editing directly on Pi will be undone by the next sync.
